# タスクサイクルの詳細手順

このドキュメントは、タスクを進める際の反復サイクルの詳細を定義します。

---

## サイクルの概要

```
計画の言語化 → 詳細化（仕様検討）→ 実行（実装）→ 検証 → 振り返りを記録
```

各フェーズを順番に実施し、必要に応じて前のフェーズに戻ります。

---

## フェーズ 1: 計画の言語化

**目的**: 取り組むタスクの目的と範囲を明文化する。

**手順**:
1. タスクの目的を1〜2文で記述する
2. 完了の定義（完了条件）を箇条書きで列挙する
3. `docs/masterplan.md` にタスクを追加し、ステータスを「未着手」にする
4. タスクテンプレート（[`docs/workflow/templates/task.md`](templates/task.md)）をコピーしてタスクファイルを作成する

**チェックポイント**:
- [ ] 完了条件が具体的で検証可能か
- [ ] 他のタスクとの依存関係が明確か

---

## フェーズ 2: 詳細化（仕様検討）

**目的**: 実行に必要な仕様・制約・前提条件を明確にする。

**手順**:
1. 技術的な制約・前提条件を洗い出す
2. 実装方針・調査方針を決める
3. 不明点があれば調査してこのフェーズ内で解消する
4. `docs/masterplan.md` のタスクステータスを「進行中」に更新する

**チェックポイント**:
- [ ] 実行に必要な情報がすべて揃っているか
- [ ] スコープが明確で膨らみすぎていないか

---

## フェーズ 3: 実行（実装）

**目的**: 計画・仕様に従って成果物を作成する。

**手順**:
1. 小さな単位で実装・執筆を進める
2. 進捗を随時タスクファイルに記録する
3. 問題が発生したらフェーズ 2 に戻って仕様を見直す

**チェックポイント**:
- [ ] 完了条件を意識して実装しているか
- [ ] 不要な変更範囲の拡大をしていないか

---

## フェーズ 4: 検証

**目的**: 成果物が完了条件を満たしているかを確認する。

**手順**:
1. フェーズ 1 で定義した完了条件を1つずつ確認する
2. 問題が見つかればフェーズ 3 に戻って修正する
3. すべての完了条件を満たしたら振り返りフェーズへ進む

**チェックポイント**:
- [ ] すべての完了条件が満たされているか
- [ ] 意図しない副作用・変更が発生していないか

---

## フェーズ 5: 振り返りを記録

**目的**: 学んだことや気づきを次のタスクに活かせる形で記録する。

**手順**:
1. うまくいったこと・改善できること・学んだことを記録する
2. `docs/masterplan.md` のタスクステータスを「完了」に更新し、成果物リンクを追加する
3. 必要に応じて `docs/ai-guidelines-master.md` とサービス向けファイルを更新する

**チェックポイント**:
- [ ] 振り返りが記録されているか
- [ ] `docs/masterplan.md` が最新状態に更新されているか
- [ ] ガイドラインを更新した場合、全サービス向けファイルに反映したか

---

## テンプレート

新しいタスクを開始するときは [`docs/workflow/templates/task.md`](templates/task.md) を使用してください。
